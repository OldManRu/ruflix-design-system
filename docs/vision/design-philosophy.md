# Design Philosophy

RuFlix exists to make spending time with personal media feel effortless, beautiful, and comfortable.

The product is not trying to become a general-purpose developer platform or a feature-heavy media dashboard. It is an environment for watching and enjoying movies, TV shows, live TV, and family media with as little friction as possible.

## North Star

Create the most comfortable place to enjoy personal media.

## Principles

### 1. Media is the hero

Artwork, video, posters, thumbnails, and personal memories should carry the experience. Interface elements should support the content without competing with it.

### 2. Comfort beats density

RuFlix should prefer breathing room, readable typography, and relaxed pacing over information density. The interface should be enjoyable from a couch, desk, tablet, or shared family screen.

### 3. Intuitive before powerful

A feature that requires explanation should be simplified. RuFlix should feel obvious to family members, friends, and self-hosting users without needing training.

### 4. Alive, not busy

Subtle ambient motion, smooth transitions, and artwork-driven backgrounds should make the interface feel alive. Animation should never distract from choosing or watching media.

### 5. Progressive disclosure

Show only what is needed first. Reveal metadata, actions, and deeper information as users focus, hover, open details, or ask for more.

### 6. Every feature earns its place

Just because RuFlix can do something does not mean it should. Features should make watching media easier, calmer, or more enjoyable.

## Product Litmus Test

Before adding a feature, ask:

- Does this help users enjoy their media?
- Does this make the experience calmer instead of busier?
- Would someone miss it if it disappeared?
- Is there a simpler way to accomplish the same goal?
- Would this still feel worth maintaining two years from now?

If the answer is unclear, simplify or defer the feature.
