# Motion

RuFlix motion should make the interface feel alive without becoming busy.

## Motion Principle

Users should notice motion only when it is missing.

## Approved Motion Patterns

- Gentle focus scale on selected cards
- Soft elevation transitions
- Smooth shelf scrolling
- Ambient background fades driven by artwork
- Player controls fading in and out
- Subtle page transitions

## Avoid

- Flashing elements
- Constant pulsing
- Aggressive autoplay previews
- Busy animated backgrounds
- Motion that distracts during playback

## Timing Guidance

| Pattern | Duration |
| --- | ---: |
| Focus lift | 120–180ms |
| Button hover/focus | 100–150ms |
| Shelf movement | 180–260ms |
| Page transition | 220–320ms |
| Ambient background transition | 500–1000ms |
| Player chrome fade | 180–240ms |

## Accessibility

Respect reduced motion preferences. Reduced motion should preserve clarity while removing non-essential movement.
