# Media Details

The Media Details screen is where a user decides whether to press Play.

## Purpose

Provide enough confidence to start watching without turning the page into a database view.

## Design Intent

Media Details should feel cinematic, spacious, and personal. It should inherit the atmosphere of the selected media through backdrop artwork, ambient color, and soft depth.

## Primary Content

- Title or logo treatment
- Year
- Rating
- Runtime
- Genre
- Short description
- Resume or Play action
- Trailer action when available
- More actions where needed

## Secondary Content

Secondary content should appear below the main decision area.

- Continue Watching state
- Cast
- Extras
- Related media
- Technical information
- Audio and subtitle options

## Progressive Disclosure

Initial view should focus on:

1. Artwork
2. Title
3. Essential metadata
4. Play / Resume
5. Short description

Deeper metadata should be available but visually secondary.

## Action Priority

1. Resume or Play
2. Trailer
3. Add to Favorites / Watchlist
4. More

## Metadata Guidance

Use readable metadata pills or compact groups instead of dense text strings where appropriate. Metadata should be scannable from couch distance.

## Cast

Cast should be visual and horizontal. Actor imagery should help users recognize familiar people without turning the screen into a research page.

## Extras

Extras should appear only when available. Examples:

- Behind the scenes
- Trailers
- Featurettes
- Interviews
- Deleted scenes

## Comfort Rules

- Avoid overwhelming the first viewport.
- Do not force users to scroll before they can press Play.
- Keep primary actions large and obvious.
- Keep secondary actions calm.
- Use artwork atmosphere without reducing readability.
